## 一、概述

**小米传送门**（内部代号 **Taplus**）是 Xiaomi/HyperOS 系统内置的一款智能内容识别工具。用户在屏幕上**长按**文字或图片时，传送门会自动识别内容并提供相关操作（如翻译、搜索、购物链接、淘口令识别、二维码识别等），同时支持跨设备剪贴板同步（Mi Continuity）。

| 维度 | 说明 |
|------|------|
| 包名 | `com.miui.contentextension` |
| 版本 | 4.1.3 (SNAPSHOT, 编译日期 250917) |
| 架构 | Kotlin + Java 混合开发，使用 AndroidX 和 MIUIX 组件库 |
| 混淆 | 未混淆（`com.miui.contentextension` 所有类名、方法名完整保留） |
| DEX 大小 | 5.6 MB（单 dex） |
| 总源码量 | 5,744 个 smali 文件（含依赖库） |
| 原生代码 | 无 `.so` 原生库 |
| 最低 SDK | Android 11 (API 30) |
| 目标 SDK | Android 14 (API 34) |

### 3.2 主服务启动 (`TextContentExtensionService`)

系统通过两种方式启动该服务：

1. **系统广播/Intent 触发**：当用户长按屏幕时，MIUI 系统发送 Intent 启动服务
2. **剪贴板数据到达**：携带 `clipboard_data` extra 的 Intent 启动

**服务生命周期**：
```
onCreate()
├── AdaptUtils.startForeground()       # 启动前台服务（通知栏常驻）
├── super.onCreate()
└── onInit()

onInit()
├── 创建 AppStateWatcher               # 监控当前前台 Activity
├── 创建 UserChangedReceiver           # 多用户切换监听
├── 创建 HomeWatcherReceiver           # 主页键监听（返回桌面时自动关闭）
├── 创建 KeyboardWatcherReceiver       # 键盘弹出/隐藏监听
├── EventBus.getDefault().register()   # 注册 EventBus
├── 监听 ConfigDataKVPref 变化
└── launchConfigLoad()                 # 启动配置加载

onStartCommand(intent)
├── 检查 mIsInit，未初始化则调用 onInit()
├── 如果有 "clipboard_data" extra
│   └── startClipboardTask(text)       # → 剪贴板浮窗
└── 否则
    └── registerContentListener()      # → 文本拾取模式

onDestroy()
├── EventBus.unregister()
├── ContentCatcherManager.unregisterContentListener()
├── destroyFloatView()                 # 销毁所有悬浮窗
└── 停止所有监听器
```

### 3.3 自停止机制

服务使用 `StopSelfHandler` 和 `scheduleStopSelfJob()` 实现智能自停止：
- 任务完成后延迟 60 秒（`60000ms`）自停止
- 新任务到达时取消之前的自停止计划
- **目的**：节省系统资源，不长时间占用后台

## 四、核心业务逻辑

### 4.1 内容拾取流程（长按触发）

这是传送门最核心的功能——用户长按屏幕任意内容时的处理流程。

```
用户长按屏幕
      │
      ▼
[MIUI 系统] ContentCatcherManager 捕获操作
      │
      ▼ 回调 IContentListenerCallback
registerContentListener() 收到回调
      │
      ▼
onStartPick()
      │
      ├─ 检查：PC 模式？ → 直接返回
      ├─ 检查：正在测试中？/任务未完成？/键盘弹出？ → 跳过
      ├─ 检查：折叠屏小屏？ → 直接返回
      ├─ 检查：传送门总开关关闭？ → 直接返回
      ├─ 检查：observe_control_event 事件？ → 直接返回（系统控制操作）
      ├─ 检查：隐私政策未更新？ → 弹窗要求同意
      ├─ 检查：剪贴板隐私弹窗？ → 弹窗要求同意
      │
      ▼ 所有检查通过
解析 ClientCatcherResult（内容捕获结果）
      │
      ├─ 文本模式 (sIsTextMode = true)
      │   ├─ 提取选中文本
      │   ├─ 获取来源包名/Activity/触摸点坐标
      │   └─ → startPickTextTask()
      │
      ├─ 图片模式 (sIsOtherMode = true)
      │   ├─ 获取截图 Bitmap
      │   └─ → startPickImageTask()
      │
      └─ 其他模式
          └─ 交由子模块处理
```

### 4.2 文本模式处理

```

      │
      ▼
保存选中文本 → sContent
保存来源信息 → sInjectorPackage / sInjectorActivity / sInjectorPoint

      │
      ▼
createSplashFloatView()         # 显示"启动动效"悬浮窗
      │
      ▼
启动 TextRecognitionTask（异步）
  ├─ 文本分词 (SegmentUtils)
  │   ├─ 中文分词（内置词典 + 模型）
  │   ├─ 英文分词
  │   └─ 数字/特殊格式识别
  │
  └─ 推荐查询 (QueryRecommend / TextRecommendationUtils)
      ├─ 本地词典匹配（商圈/人名/公司名等）
      └─ 云端推荐请求 (TaplusNetworkClient)
          ├─ 购物推荐（淘宝/京东商品识别）
          ├─ 翻译（中译英/英译中）
          ├─ 百科查询
          ├─ 地址/电话识别
          └─ 电商口令识别（淘口令等）
      │
      ▼
准备就绪 → 通知 RecommendationMonitor
      │
      ▼
onSegmentReady() / onDetailReady()
      │
      ▼
关闭 SplashFloatView → 显示 TaplusMainFloatView
  ├─ 选中文本高亮显示
  ├─ 推荐结果卡片列表
  │   ├─ 商品卡片
  │   ├─ 翻译卡片
  │   ├─ 百科卡片  
  │   └─ 复制/分享快捷操作
  └─ 拖拽选择区域调整
```

### 4.4 悬浮窗系统

传送门的 UI 全部通过 **系统悬浮窗**（`WindowManager` + `TYPE_APPLICATION_OVERLAY`）实现，不依赖 Activity。

#### 悬浮窗层级

| 类型 | 类名 | 用途 |
|------|------|------|
| 动效悬浮窗 | `TaplusSplashFloatView` | 触发时的水波纹/动画效果（161KB smali，最复杂的 UI 组件） |
| 主悬浮窗 | `TaplusMainFloatView` | 展示识别结果和推荐操作 |
| 分组视图 | `TaplusMainGroupView` | 推荐结果分组显示 |

#### 生命周期

```
createFloatView()
  → BaseFloatView 构造
      ├─ 获取 WindowManager 服务
      ├─ 配置 LayoutParams
      │   ├─ type = TYPE_APPLICATION_OVERLAY (2038)
      │   ├─ flags = FLAG_NOT_FOCUSABLE | FLAG_LAYOUT_IN_SCREEN
      │   └─ layoutInDisplayCutoutMode = 1 (短边)
      └─ inflate 布局

  → addToWindow()
      └─ mWindowManager.addView(mMainView, wmParams)

destroyFloatView()
  → removeFromWindow()
      └─ mWindowManager.removeViewImmediate(mMainView)
```

#### 主悬浮窗结构 (`TaplusMainFloatView`)

```
TaplusMainFloatView
├── 背景阴影 (mShadow)
├── 截图预览 (mScreenshotView) — PartialScreenshotView
│   └── 选定区域高亮 (ShapeFactory)
├── 识别结果卡片 (mRecognitionCard)
│   └── TaplusRecognitionExpandedCard
├── 推荐列表 (mRecommendList)
│   └── RecyclerView + TaplusRecommendAdapter
│       ├── 商品推荐卡片
│       ├── 翻译结果卡片
│       └── 更多操作卡片
├── 拖拽图标 (mIconDrag)
└── 主卡片组 (mMainCardGroup) — TaplusMainGroupView
```

### 4.5 剪贴板模块

剪贴板模块处理系统剪贴板的读取、监控和跨设备同步。

#### 剪贴板广播接收 (`ClipboardReceiver`)

```
onReceive()
  └─ Intent Action = "miui.intent.action.CLIPBOARD" 或系统剪贴板变化
      │
      ├─ 剪贴板内容截断（超过 5000 字符截断）
      ├─ 保存描述时间戳 (mDescriptionTimestamp)
      ├─ 触发 ClipboardMonitor 检查
      └─ 发送到 TextContentExtensionService
          └─ → startClipboardTask() → 显示剪贴板悬浮窗
```

#### 剪贴板监控 (`ClipboardMonitor`)

观察者模式，三类监听器：
- `OnDeviceReadyListener` — 设备就绪状态
- `OnNerReadyListener` — NER（命名实体识别）就绪
- `OnCopyRecommendReadyListener` — 复制推荐就绪

#### 剪贴板编辑 (`EditActivity`)

独立的 Activity，允许用户查看和编辑剪贴板历史。

### 4.6 跨设备同步（Mi Continuity）

#### Mi Continuity 服务

```
MiContinuityService (extends Service, implements ChannelListener)
  ├─ 创建 ContinuityChannelManager
  ├─ 注册 ServiceName = "ContentExtensionService"
  └─ 在 ChannelListener 回调中处理跨设备剪贴板同步
      ├─ onChannelActive()      — 通道激活
      ├─ onMessageReceived()     — 接收其他设备剪贴板
      └─ sendClipboardData()     — 发送剪贴板到其他设备

StaticMiContinuityService
  └─ 静态配置版本的 Continuity 服务（开机自动注册）
```

#### Miui+ 集成

```
MiuiPlusReceiver
  └─ 接收 Miui+（小米妙享）的剪贴板同步广播
      └─ 解析内容 → 转发到 TextContentExtensionService
```

#### MiLink 集成

```
MiLinkSwitchBroadcastReceiver
  └─ 监听 MiLink 连接状态变化，启用/禁用 Continuity 功能
```

## 五、数据与网络层

### 5.1 网络栈

使用 **OkHttp** 作为 HTTP 客户端，基于 RxJava 响应式编程。

```
TaplusNetworkClient (单例)
  └─ OkHttpClient (配置: 连接超时/拦截器/缓存)
      ├─ TokenManager — OAuth 令牌管理
      │   ├─ 获取 Access Token
      │   ├─ Token 刷新
      │   └─ 存储 Token
      ├─ HostConfig — 服务器地址配置
      │   ├─ 正式环境 (api.huami.com?)
      │   └─ 测试环境
      ├─ RequestParamsUtil — 请求参数构建
      │   ├─ 通用参数（设备信息/版本号等）
      │   └─ 签名生成
      └─ JsonRequest — JSON 请求封装
          └─ 解析响应 → 推荐结果
```

### 5.2 API 接口

通过 `ApiInterface` 定义的 API：

| 接口 | 功能 |
|------|------|
| 文本推荐 | 对选中文本进行商品/翻译/百科等推荐 |
| OCR 识别 | 图片文字识别请求 |
| 配置下载 | 云端黑白名单、商圈规则更新 |
| 分析上报 | 用户行为数据上报（OneTrack） |

### 5.3 文本识别与推荐引擎

```
文本输入
    │
    ▼
SegmentUtils (分词)
  ├─ 中文分词（assets 词典: pinyin.txt, core.dic, person.dic 等）
  ├─ 英文分词（englishLibrary.dic）
  ├─ 数字/特殊格式（numberLibrary.dic）
  ├─ 命名实体识别（nature.table, nature.map, company.data 等）
  └─ 繁简转换支持（simp.txt, trad.txt, simplified.txt）
    │
    ▼
Entity 提取（实体识别）
  ├─ 商品名
  ├─ 地址/电话
  ├─ 人名
  ├─ 公司名
  └─ 电商口令
    │
    ▼

  ├─ 本地规则匹配（commerce_rules.json — 电商规则）
  └─ 云端推荐查询
      └─ → TaplusNetworkClient → 返回 → TextRecommendationInfo
```

## 六、配置管理

### 6.1 `ContentExtensionConfig`

应用核心配置管理器，管理每应用每 Activity 的详细配置：

```
ContentExtensionConfig (单例)
├── mConfigList: Map<packageName, AppConfigInfo>
│   └── AppConfigInfo
│       ├── type: int (1=文本, 2=图片, 3=混合)
│       ├── trigger: [singlePickMode, doublePickMode]
│       └── configList: [ActivityConfigInfo...]
│           └── ActivityConfigInfo
│               ├── activityName
│               └── validArea: [{validAreaTop, validAreaBottom}...]
├── mBlackList: List<String>  (黑名单包名)
└── mIsForbiddenByUser: Boolean (用户禁用)
```

**配置来源**（按优先级）：
1. 云端配置（通过 HTTP 下载）
2. Assets 内置配置（`assets/config.json`）
3. 本地默认配置

### 6.2 `config.json` 示例

从内置的 `assets/config.json` 可见，传送门针对主流应用有定制配置：

- **微信 (`com.tencent.mm`)**: 聊天界面排除状态栏/输入法区域（top 0-10%, bottom 93-100%）
- **有道翻译 (`com.youdao.translator`)**: 全屏可用，从顶部 90% 开始
- **唱吧 (`com.changba`)**: 全屏可用
- **默认应用**: 单指/双指模式都支持

### 6.3 黑白名单

- **黑名单**: `assets/blacklist.json` + 云端动态更新
- **白名单**: 通过 `SwitchControlProvider` 提供给系统设置
- **运行时更新**: `PackageChangeReceiver` 监听应用安装/卸载，自动更新规则
